# 統合実装計画

## 目的
競馬AIアプリの分散していた実装計画を一本化し、今後はこのファイルだけを更新する。

## 現在の到達点
- プロジェクトは `keiba_project/` 配下へ再編済み
- netkeiba 出馬表・オッズ・人気の取得基盤を導入済み
- バックエンドに Agent Manager 構成を導入済み
- UI は買い目生成中心から馬評価中心へ移行済み
- ユーザー履歴を使う DNA 分析と simulation 画面を導入済み
- PWA / Render 運用の基盤を整備済み

## 実装の変遷サマリー

### 1. 初期段階: 買い目自動生成
- 過去成績ベースで推奨買い目を出す Web アプリを構想
- モックデータによる次レース表示、買い目カード、購入金額集計を設計

### 2. ライブデータ対応
- netkeiba から出馬表・オッズ・人気を取得するスクレイピングを導入
- `live_data.json` 保存や URL 正規化で取得精度を改善

### 3. ロジック高度化
- 人気順依存を減らし、妙味・実力・荒れ判定を使う戦略ロジックへ更新
- 穴馬候補、券種別優先順位、戦略分岐を導入

### 4. プロジェクト再編と履歴分析
- 分散フォルダを統合し、`analysis/` と `simulation` 画面を追加
- CSV に馬名・人気・オッズ列を拡張し、履歴分析の土台を整備

### 5. Agent Manager 化
- スコアリング、戦略生成、最終統合をバックエンドの各 Agent に分離
- `/api/predict` でスクレイピングから予測返却まで一体化

### 6. 馬評価特化ツール化
- 買い目生成を主役から外し、馬評価カード中心 UI に刷新
- AI コメント、近走成績、上がり、DNA 一致を表示
- 穴馬選出ロジックは中穴重視へ継続調整

## 現在の主要コンポーネント

### バックエンド
- `backend/main.py`
  - スクレイピング、近走取得、ユーザープロファイル取得、予測 API を担当
- `backend/agents/scoring_agent.py`
  - 妙味、実力、安定度、DNA、穴馬スコアを計算
- `backend/agents/strategy_agent.py`
  - レース条件分析と戦略組み立てを担当
- `backend/agents/manager.py`
  - 本命、対抗、穴馬、おすすめ馬の最終分類と AI コメント生成を担当
- `backend/database.py`
  - 履歴 CSV の取り込みと DB 初期化を担当

### フロントエンド
- `app/index.html`
  - メイン画面
- `app/js/app_v15.js`
  - バックエンド予測結果の描画
- `app/simulation.html`
  - 履歴読み込みと DNA 学習導線
- `app/js/simulation_v5.js`
  - simulation 画面の集計と送信

### データ
- `analysis/馬券投票履歴_enriched.csv`
  - 学習用の拡張済み履歴
- `analysis/enrich_csv.py`
  - 履歴 CSV の補完スクリプト

## 今後の更新方針

### 優先度高
- 馬評価 UI とスコアリングの精度改善を継続
- 穴馬選出と DNA おすすめの精度調整

### 優先度中

- API / DB 周りの永続化強化
- モバイル表示と PWA キャッシュ制御の改善

### 優先度低
- LLM を使う説明文生成の高度化
- エージェント分割の追加

- Render 上で主要画面と `/api/predict` が安定動作する
- 馬評価、本命・対抗・穴馬・おすすめが自然に表示される
- simulation から学習した傾向が予測に反映される
- このファイルの内容が最新状態を反映している

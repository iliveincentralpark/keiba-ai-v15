# 買い目生成ロジック改善 — 修正確認レポート

## 実施した変更

### 1. `scoring_agent.py` — 実力スコア・穴馬スコア強化

| 変更点 | 改善前 | 改善後 |
|--------|--------|--------|
| 実力スコア加重 | `max*0.7 + last*0.9` の最大値 | 直近×0.5 + 平均×0.3 + 最大×0.2（直近重視） |
| 指数0時のデフォルト | `0.88`（過大評価） | `0.75`（実力不明馬を抑制） |
| stability | 人気のみ反比例 | 人気反比例 + ability_scoreが高い馬にボーナス |
| 穴馬スコア | 未実装 | `upset_score`：人気6位以下かつオッズ20倍以上・実力ある馬に付与 |

### 2. `strategy_agent.py` — 4券種・おすすめ順・穴馬選出

- デフォルト買い目が **3種 → 4種** に拡張
- レース条件別の優先順位切り替え：

  | 条件 | ◎推奨 | ○ | △ | ☆ |
  |------|-------|---|---|---|
  | 本命突出型 | 馬単 | 3連単 | 3連複 | 馬連 |
  | 低オッズ型 | ワイド | 3連複2頭軸 | 馬連 | 3連複1頭軸 |
  | 穴馬ありの接戦型 | 3連複 | ワイド | 3連単 | 馬連 |
  | 標準 | 3連複 | 馬連 | ワイド | 3連単 |

- ワイドBOXの相手に穴馬候補（`upset_score`上位）を自動混入
- 各betに `priority`・`priority_label` フィールドを追加

### 3. `manager.py` — 본命・対抗・穴馬分類

`horse_roles` をレスポンスに追加：
- `honmei`：スコア1位
- `taikou`：スコア2〜3位
- `ana`：`upset_score > 0` の馬を降順に最大2頭

### 4. `app_v15.js` — UI表示の追加

- **AIピックアップパネル**（本命◎ / 対抗○ / 穴馬🎲）を買い目カードの上部に表示
- 買い目カードのタイトルに **おすすめ順バッジ**（◎推奨／○安定／△妙味／☆配当）を追加
- ワイドBOX等の相手馬に穴馬候補は **オレンジ枠 + 🎲マーク** で強調表示
- スコアテーブルに「穴」列追加

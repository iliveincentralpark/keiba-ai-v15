# 次回実装計画：WEBアプリのスクレイピング連携とモバイル対応

## 第一フェーズ：新しいレコードのスクレイピング連携の仕様
過去のPythonスクリプト（`enrich_csv.py`）で実現していた netkeiba からの情報取得を、ユーザーの入力やWEBアプリ操作に基づいてリアルタイムまたはバッチで実行する機能への拡張。

### 実装要件
1. **フロントエンドからの取得命令**
   - ユーザーが新しいレース情報やURLを入力した際、自動的に補完データ（馬名・人気・オッズなど）を取りに行く仕組み。
2. **バックエンド（API）またはAWS Lambda/Cloud Functions**
   - Pythonでのスクレイピングスクリプトが安定している場合、軽量なバックエンドAPI（Flask/FastAPI、またはNode.js Express）を立ち上げ、フロントエンドと通信させる構成を検討する。
3. **データストレージの再考**
   - 現在のCSVから、必要であればSQLite等軽量なDBまたはバックエンド管理データとして移行することも将来的に検討する。

## 第二フェーズ：モバイル・マルチデバイス対応（ローカル環境からの脱却）
スマートフォン等で外出先からも結果を確認・操作できる仕様の構築。

### 実装要件
1. **ホスティング環境の選定とデプロイ**
   - サーバーサイドのスクレイピングを伴うため、Render（無料枠のバックエンド+フロント）または Vercel（サーバーレスFunctions有）を利用し、プロジェクトをデプロイする。
   - ※スクレイピングのブロック（IP制限など）を回避するため、稼働環境に適した実装の調整が必要。
2. **UI/UX のレスポンシブ化**
   - PC（ローカル）でのシミュレーション画面や管理画面だけでなく、スマートフォン（縦型画面）で表や入力フォームが快適に使用できるようにCSSフレームワークや独自メディアクエリで見直す。
3. **PWA（Progressive Web App）化の検討**
   - スマホのホーム画面に追加でき、ネイティブアプリライクな利用体験をもたらすPWA化の実装。

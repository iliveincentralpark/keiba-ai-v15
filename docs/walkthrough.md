# 統合作業ログ

## 概要
本プロジェクトでは、競馬予想アプリをモック中心の買い目生成ツールから、実データ連動の馬評価アプリへ段階的に進化させてきた。分散していた作業ログをこのファイルに統合する。

## 主要な進化の流れ

### 1. 初期版の構築
- 過去の勝ちパターンをもとに買い目を自動生成する Web アプリを実装
- UI 上で推奨理由や合計購入金額を確認できる構成を整備

### 2. スクレイピング導入
- netkeiba から出馬表、単勝オッズ、人気を取得する仕組みを追加
- 取得結果をローカル保存できるようにし、リアルデータ利用へ移行

### 3. URL 正規化と取得精度改善
- 特集ページなどから正規の出馬表 URL へ自動変換する処理を導入
- データ欠損や文字切れのあるケースを減らし、取得品質を向上

### 4. 買い目ロジックの強化
- 人気順ベースから、妙味・実力・荒れ判定を加味するロジックへ更新
- 穴馬候補、券種別優先順位、戦略分岐を導入

### 5. プロジェクト再編
- 分散していた関連コードやドキュメントを `keiba_project/` に集約
- `analysis/` と `docs/archive/` を整理し、履歴分析系の基盤を整備

### 6. 履歴分析と DNA 機能
- CSV に馬名・人気・オッズを補完
- simulation 画面から過去履歴を取り込み、人気帯や買い方の傾向を学習

### 7. Agent Manager 導入
- スコアリング、戦略生成、最終統合をバックエンド Agent に分離
- `/api/predict` を追加し、フロント側ロジックを軽量化

### 8. 馬評価特化 UI への刷新
- 買い目中心 UI を縮小し、馬評価カード中心の表示に変更
- 本命、対抗、穴馬、おすすめ馬をサマリー表示
- AI コメントを自然文ベースへ刷新

### 9. 近走データと穴馬ロジック改善
- `shutuba_past.html` から近走着順と上がりを一括取得
- 中穴が落ちやすい問題を継続修正
- 直近では欠損値除外、中穴優先、超人気薄の抑制を実施

## 現在の状態
- 予測 API はバックエンド主導
- UI は馬評価中心
- DNA によるおすすめ表示あり
- 穴馬ロジックは中穴帯を優先する形に調整済み
- GitHub `main` へ最新調整を反映済み

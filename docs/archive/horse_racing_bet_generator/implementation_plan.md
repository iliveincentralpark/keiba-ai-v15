# 競馬買い目自動生成Webアプリ 実装計画 (Horse Racing Bet Generator)

Pythonで過去の競馬成績データ(プロファイル)から抽出した「勝ちパターン」を元に、次レース情報を取得・予測して自動的に買い目を生成するWebアプリケーション(SPA)を構築します。ユーザーが推奨理由を理解し、IPATへ連携して即座に購入できるよう、モダンでリッチなUI（ダークモード、グラスモーフィズム、アニメーション等採用）を提供します。

## User Review Required
- 現在、リアルタイムで次レースの情報を取得するAPI(JAT等)は認証や有料契約が必要なケースが多いため、今回は**「架空の次レースデータ（馬名・オッズ・荒れ予測など）を生成するモックシステム」**をフロントエンド内に構築し、そのデータに対して分析プロファイルを適用する形でよろしいでしょうか。
- デザインにはTailwindを使用せず、要件に応じたVanilla CSSとCSS Modulesを用いて、プレミアムで動的なルック＆フィール（グラス効果、滑らかなトランジション）を構築します。

## Proposed Changes

### 1. Webアプリケーション基礎
プロジェクトルート(`/Users/kojimarei/Desktop/Antigravity/horse_bet_web`)に Vite + React + TypeScript 環境をセットアップします。

### 2. コアロジック (TypeScript)
#### [NEW] `src/lib/betLogic.ts`
- `analyze.py` で定義した「勝ちパターン」の買い目生成ロジック（ワイドBOX, 馬連1頭軸, レース波乱度に応じた3連複）をTSへ移植。
- モックデータに基づき、最適化された買い目配列（券種、点数、フォーメーション内容、過去実績テキスト）を返却する関数。

#### [NEW] `src/lib/mockData.ts`
- 本日開催の次レース（例：中山11R 有馬記念など）を出力するモックジェネレーター。

### 3. UI・デザイン
#### [NEW] `src/index.css` / `src/App.css`
- ダークトーンを基調とし、鮮やかなアクセントカラー（ネオンブルー、ゴールド）を用いたリッチなスタイルシート。

#### [NEW] `src/App.tsx`
- メインレイアウト。ヘッダー（レース概要・荒れ予測）、生成された買い目カード一覧、下部の購入サマリーを配置。

#### [NEW] `src/components/BetCard.tsx`
- 券種ごとに「過去実績（回収率）」「選択した馬番・馬名」「点数・金額」を表示するカードコンポーネント。

# 競馬買い目自動生成Webアプリ 実装完了報告

過去の馬券成績データに基づく「勝ちパターン（プロファイル）」を活用し、次レースの推定データに対して最適な買い目を自動生成するWebアプリケーションの実装が完了しました。

## 実装した機能・システム
- **プロファイルに基づくアルゴリズム構築** (`js/app.js`)
  - **ワイドBOX戦略**: 過去回収率445%、的中率100%の最強パターンを軸評価上位の5頭へ適用。
  - **馬連1頭軸戦略**: 過去回収率420%。本命の信頼度が高いレースの際に、点数を絞って的確に狙う。
  - **3連複の可変点数戦略**: レースの「荒れ予測」パラメータを自動判定し、波乱含みなら「1頭軸流し（多点数）」、堅実な場合は「2頭軸流し（超少点数）」へと買い目をダイナミックに変更。
- **プレミアムUIコンポーネント** (`index.html`, `css/style.css`)
  - ユーザーがIPATで購入する前に「なぜこの買い目なのか」を直感的に理解できるよう、各買い目カードに「💡推奨理由（過去の回収率や意図）」を配置しました。
  - デザインはダークテーマを基調とし、透明感のあるグラスモーフィズム（透過パネル+すりガラス効果）やグラデーションを用いて、洗練された没入感のある体験を提供します。
  - 券種ごとに左のボーダー色を変え、視認性を高めています。
- **インタラクティブな金額計算**
  - 生成された推奨買い目に対して「ON/OFF」のトグルボタンを実装し、ユーザーの選択に応じて下部の合計購入金額が即座にリアルタイム更新されるUXを実現しました。

## 動作確認 (Verification)
ブラウザ・サブエージェントを通じたE2Eテストにて、以下の動作を検証・確認しました。

1. **初期描画の確認**
   モックデータ (`mock.json`) をロードし、「中山11R 有馬記念」のレース情報がヘッダーに描画されていること。
2. **ロジックの適用確認**
   荒れ予測が `true` と定義されたモックデータに対し、ロジック通り「ワイドBOX」「馬連1頭軸流し」「3連複**1頭軸**流し（多点数）」の3パターンが正しく推奨されたこと。
3. **インタラクションの確認**
   初期合計金額 ¥4,600 の状態から、「ワイド BOX (¥1,000)」のトグルボタンを一度クリック（OFF化）したところ、即座に合計金額が ¥3,600 に再計算されて表示されることを確認しました。

# AI買い目ジェネレーター - 実行・運用マニュアル

## 1. アプリケーションの構成
本ツールは、大きく分けて以下の2つのプロセスで動作しています。
1. **フロントエンド (Web UI)**: ユーザーが操作する画面。HTML/CSS/JS で構築され、`http.server` (ポート8000) で配信されます。
2. **バックエンド API (Flask)**: netkeibaのURLを受け取り、最新の出馬表・オッズデータをスクレイピングしてフロントエンドに返すPythonサーバー (ポート5000)。

## 2. 起動手順 (使い方)

アプリを利用するには、**2つのターミナル（コンソール画面）**を開き、それぞれでサーバーを立ち上げる必要があります。

### ステップ1: バックエンドAPIの起動
1. ターミナルを開きます。
2. アプリのディレクトリに移動します:
   ```bash
   cd /Users/kojimarei/Desktop/Antigravity/horse_bet_web
   ```
3. 以下のコマンドでFlaskサーバーを起動します:
   ```bash
   python3 app.py
   ```
   > ターミナルに `* Running on http://127.0.0.1:5000` と表示されたまま待機状態になれば成功です。このターミナルは閉じないでください。

### ステップ2: フロントエンド(Web UI)の起動
1. **別の新しいターミナル**を開きます（またはターミナルの「新しいタブ」などを開きます）。
2. 同じくアプリのディレクトリに移動します:
   ```bash
   cd /Users/kojimarei/Desktop/Antigravity/horse_bet_web
   ```
3. 以下のコマンドでローカルWebサーバーを起動します:
   ```bash
   python3 -m http.server 8000
   ```
   > ターミナルに `Serving HTTP on :: port 8000` のように表示されれば成功です。

### ステップ3: アプリの利用
1. ブラウザ (ChromeやSafariなど) を開き、以下のURLにアクセスします。
   **[http://localhost:8000](http://localhost:8000)**
2. 画面上部にある入力欄に、これから購入したい netkeiba の出馬表URL (例: `https://race.netkeiba.com/race/shutuba.html?race_id=...`) をペーストします。
3. **「最新オッズ取得」** ボタンをクリックします。
4. 数秒待つと、「オッズ再計算が完了しました」というアラートが出現し、画面のレース名や、各買い目（10点など）に「馬名」と「(何番人気 何倍)」という最新情報が反映され、トリガミなどが判断しやすくなります。

## 【開発・実装履歴の保存】（次回作業用引き継ぎメモ）

### これまでに実装したこと (〜2026/02/22)
1. **プロファイルベースの買い目生成ロジック (`js/app.js`)**
   - 過去成績CSVから算出した勝ちパターンに基づくアルゴリズム（ワイドBOX回収率445%など）を3パターン搭載。「波乱/堅実」の予測フラグによって3連複の点数をダイナミックに切り替える処理。
2. **プレミアムUIの実装 (`index.html`, `css/style.css`)**
   - グラスモーフィズムを用いたダーク基調の洗練されたUI。点数に応じた合計金額のリアルタイム計算（ON/OFF機能）。
3. **ライブオッズ取得機能 (今回実装)**
   - `scraper.py` を用いて netkeiba の指定URLの HTML テーブルをパースし、馬番・馬名・オッズ・人気情報を抽出。
   - `app.py` でそれをローカルの Flask API として配信 (`/api/scrape?url=...`)。
   - フロントエンド (`app.js` の `fetchLiveOdds` および `mergeOdds` 関数) でこれを叩き、静的なモックデータの予測結果に対して最新のオッズ情報を上書き・マージして再レンダリングする処理。

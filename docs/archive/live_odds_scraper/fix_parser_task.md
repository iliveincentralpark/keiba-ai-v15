# netkeibaスクレイパー修正タスク

## 目的
UI上で「レース名取得エラー」になる問題や、馬名が適切に更新されない（モックデータのまま、または昔のデータのまま変わらない）不具合を解消するため、`scraper.py` および `app.js` のデータマージロジックを修正する。

## 確認された不具合
- `race_info["name"]` が「レース名取得エラー」として返っている。（＝`.RaceName` というクラス名の要素が存在しないか、HTML構造が変わっている）
- 別のURLを入力しても、UI上の馬名が古いもの（阪神大賞典の馬など）に固定されてしまっている様子。

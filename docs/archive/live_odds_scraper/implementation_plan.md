# ライブオッズ・人気連動 買い目自動生成システム 実装計画

本計画は、実際の出馬表とオッズ・人気情報をスクレイピングにより取得し、「競馬買い目自動生成Webアプリ」においてリアルタイムな配当妙味を考慮した最適な買い目を推奨する拡張機能の設計・実装方針をまとめたものです。

## ゴールと背景
これまでのWebアプリはモックデータによる固定された出馬表と仮想の評価値をベースにしていましたが、これを外部のWebサイト（netkeiba等を想定）から取得した**実際の単勝オッズ・人気順**に置き換え、それらのデータに基づいて「配当的に美味しい・または堅実に押さえるべき」買い目をロジックに組み込むことを目的とします。

---

## 提案する変更内容・アーキテクチャ

システムは大きく分けて「①データ取得・生成（バックエンド的役割）」と「②表示・ロジック適用（フロントエンド）」の2層で構成します。

### 1. スクレイパーの実装 (Python)
#### [NEW] `horse_bet_web/scraper/get_odds.py`
- **目的**: 指定されたレースのURL（またはレースID）を引数として受け取り、出走馬、馬番、単勝オッズ、人気順などの情報をスクレイピングするスクリプト。
- **使用ライブラリ想定**: `requests`, `BeautifulSoup4` (場合により `pandas`)
- **出力**: 取得したデータをパース・整形し、フロントエンドが読み込める形で `horse_bet_web/data/live_data.json` としてファイル出力します。必要に応じて「荒れ予測（人気馬のオッズの割れ方などで判定）」のフラグも動的に計算します。

### 2. フロントエンドの改修 (HTML/JS)
#### [MODIFY] `horse_bet_web/index.html`
- ヘッダー付近に「データ更新日時」や「実際の単勝オッズ・人気」を表示するためのUI要素（バッジやツールチップ等）を追加。

#### [MODIFY] `horse_bet_web/js/app.js`
- **データの読み込み先変更**: `mock.json` ではなく `live_data.json` をフェッチするように変更（ファイルが存在しない場合は既存のモックへフォールバックするようエラーハンドリング）。
- **アルゴリズムの拡張**:
  - 取得したオッズ（`odds`）や人気順（`popularity`）情報を買い目生成ロジックに組み込みます。
  - *例*: ワイドBOX戦略において「1〜3番人気の組み合わせはオッズが低いため除外し、人気馬×中穴の組み合わせへスライドする」。
  - *例*: 「オッズ15.0倍〜30.0倍」の馬を穴馬として自動抽出し、3連複のヒモに組み込む。

#### [NEW] `horse_bet_web/data/live_data.json` (自動生成される想定)
- `scraper.py` によって上書き生成される、本番用のJSONデータ。

## 検証(Verification)計画

### 1. スクレイパーの動作確認 (手動)
ターミナル上でスクリプトを実行し、正常に `live_data.json` が生成されるか確認します。
```bash
python horse_bet_web/scraper/get_odds.py "https://race.netkeiba.com/race/shutuba.html?race_id=2024..."
```
※URLは直近で開催される実際のレースURLを用います。
JSONの中身を確認し、「馬名」「馬番」「単勝オッズ」「人気」が正しくパースされているかをチェックします。

### 2. フロントエンドUIおよびロジックの確認 (ブラウザサブエージェント)
ブラウザ上で開発用サーバーを立ち上げ、生成された `live_data.json` をアプリが正しく読み込んでいるか検証します。
- **オッズ表示**: 実際のオッズと人気が画面上に反映されているか。
- **ロジック反映**: 実オッズに合わせてトリガミになりそうな買い目が避けられているか、推奨理由のテキストにオッズに言及する内容（例：「単勝15倍以上の〜〜」）が含まれているか確認します。

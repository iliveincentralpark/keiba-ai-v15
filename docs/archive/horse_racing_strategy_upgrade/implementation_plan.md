# 買い目生成ロジックの刷新とデータ抽出精度の向上計画

## 現状の課題
- **データソースの不安定性**: 「特集ページ」等のURLでは、簡易API（`api_get_odds_ninki_new.html`）が使用されており、データが不完全（騎手名の文字切れ等）であったり、実際のオッズと乖離する場合がある。
- **買い目ロジックの単純化**: 現在は人気上位2〜4頭を順番に選んでいるだけで、「工夫（穴馬の選定や期待値の考慮）」が足りない。

## 修正内容

### 1. スクレイパーの強化 (scraper.py)
- **URL自動変換ロジックの導入**:
    - 特集ページのURL（`race.netkeiba.com/special/...`）などが入力された場合、自動的に正規の出馬表URL（`race.netkeiba.com/race/shutuba.html?race_id=...`）へ置換してスクレイピングを行う。
    - これにより、常に最も精度の高いデータソースを参照できるようになり、データの乖離や欠損を防止する。
- **データクレンジングの徹底**: 取得した文字列のトリミングや数値変換をさらに厳密に行う。

### 2. 買い目生成ロジックの刷新 (app.js)
- **「穴馬・期待値」選定ロジックの導入**:
    - 単なる人気順ではなく、以下の「戦略的な選定」を行うように変更：
        - **本命（◎）**: 最上位人気（1番人気など）を軸とする。
        - **対抗（◯）**: 2〜3番人気の中からオッズが最も妥当な馬を選出。
        - **相手（▲△☆）**: 
            - 4〜5番人気の「堅実な相手」に加え、
            - **6〜10番人気の中から「オッズがハネている（期待値が高い）」馬を1〜2頭自動的にピックアップ**する。
            - レースの「荒れ判定」に基づき、穴馬の含有率を動的に調整する。

## 検証プラン
- **URL変換テスト**: 特集ページのURLを入力した際、デバッグログに「正規URLに変換しました」と表示され、フルデータの抽出に成功することを確認。
- **買い目ロジックテスト**: 生成された買い目に、6番人気以下の穴馬が1頭以上含まれていることを確認。

> [!IMPORTANT]
> この修正により、ユーザーが意図しない「単なる人気順の購入」を防ぎ、AIらしい「狙い目」のある買い目を提供できるようになります。

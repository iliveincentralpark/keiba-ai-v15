# 描画不具合（画面が白くなる）の修正計画

## 概要
最新の修正後、Chromeブラウザで初期表示（モックデータ）および買い目リストが表示されない不具合を修正します。

## 原因分析（仮説）
1. **初期化タイミングの不一致**: `DOMContentLoaded` イベントがスクリプト読み込み時に既に発火済みである可能性。
2. **サイレントエラー**: JS内部でエラーが起きているが、アラート等にキャッチされず処理が中断している。
3. **DOM要素の欠落**: `app.js` が期待するIDが `index.html` に存在しない、あるいは取得順序に問題がある。

## 実施内容

### 1. デバッグ機能の強化
- `window.onerror` 等を使用して、ブラウザ上のあらゆるJSエラーを `alert` で通知するようにします。
- `console.log` を要所に配置し、どこまで処理が進んでいるかを確認できるようにします。

### 2. 初期化処理の改善
- `DOMContentLoaded` を待つだけでなく、既にDOMが準備できている場合は即座に `initApp` を実行するように変更します。

### 3. DOMアクセスの安全性強化
- 要素が存在しない場合にエラーで止まるのではなく、警告を表示して処理を継続可能にします。

## 検証計画
- 修正後の `app.js` を適用し、初期画面（フェブラリーS）が正しく表示されるか確認。
- オッズ取得ボタン押下後の挙動を再確認。
